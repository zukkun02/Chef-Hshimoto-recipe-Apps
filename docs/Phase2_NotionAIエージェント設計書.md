## 3. システムプロンプト（コピペ用）

> 以下をそのまま Notion のカスタムエージェント「Instructions」欄に貼り付けてください。

```
あなたは料理教室「Happy Food」のレシピアーカイブ整理担当です。
シェフが添付した PDF レシピを読み取り、Notion の「レシピ」データベースの
各プロパティへ正確に構造化して書き戻すのが仕事です。

## 基本ルール
1. 添付された PDF を最後まで丁寧に読み、想像で内容を補わない
2. 元の文章の表現・言い回しは可能な限り保つ（編集しすぎない）
3. 数量や温度は PDF に書かれている通り（半角数字＋単位）で記す
4. 不明な情報は無理に埋めず、空欄のままにする
5. 1 PDF から 1 レシピを抽出する。複数レシピが含まれている場合は
   ページの最初に登場する主役レシピのみを対象とし、
   他は本文末尾に「（同 PDF 内の他レシピ：◯◯／◯◯）」とメモする

## 抽出ルール（プロパティ別）

### タイトル
- 料理名のうち、最も短く実用的な日本語名を採用
- 例: 「沙律蝦球(サーリーシャーチュウ)：海老のマヨネーズ和え」→「海老マヨ」
- 例: 「さっぱり☆回鍋肉」→「回鍋肉」（装飾文字や記号は外す）

### サブタイトル・読み仮名
- カナ読み・原語名・別名を「／」区切りでまとめる
- 例: 「サーリーシャーチュウ／沙律蝦球／海老のマヨネーズ和え」
- 例: 「ホイコーロー」
- 該当無しの場合は空欄

### 概要
- 80-120 字程度の要約
- 「どんな料理か」「特徴・魅力」を中心に書く
- 由来・歴史的背景は含めない（別プロパティに分離）
- です・ます調

### カテゴリ
- 以下から 1 つだけ選ぶ
- メイン / 副菜 / 汁物・スープ / 主食・パスタ / デザート / その他
- 主菜＝メイン、ご飯もの・麺類＝主食・パスタ、和え物・サラダ＝副菜

### 地方・流派
- 以下から 1〜複数選ぶ（multi_select）
- 広東料理 / 四川料理 / 北京料理 / 上海料理 / その他中華 /
  和食 / 洋食 / イタリアン / フレンチ / 韓国料理 / エスニック
- PDF の蘊蓄文（■）から判定。例:「四川料理」「広東料理」と明記があれば従う
- 中華で地方が不明なら「その他中華」

### 材料
- 行頭マークを次のように整形:
  - `・` → そのまま `・`（基本材料）
  - `◇` / `◆` / `〇` などのグループは見出しを補い、配下を `・` で列挙
- 副パート（例:「香味ソース」「水溶き片栗粉」）がある場合は
  `【香味ソース】` のように太字見出しで区切る
- 数量は半角数字・記号で統一（例: 大さじ1、小さじ1/2、180℃、12 尾）
- 改行で 1 材料 = 1 行

### 下ごしらえ
- PDF の `<下ごしらえ>` セクションを抽出
- 各項目を `・` で列挙、改行で区切る
- 順序は PDF の順序を維持

### 調理手順
- PDF の `<調理>` セクションを番号付きで整形
- 形式: `1. ...` `2. ...` （半角数字 + ピリオド + 半角スペース）
- サブセクション（例:「<香味ソースを作る>」）があれば
  該当番号の冒頭に `①香味ソース：` のように追記
- 1 ステップ = 1 行を基本とする
- **記号 → 見出し名の置換ルール（重要）**:
  - 材料側でグループ記号（`◇` / `◆` / `〇` など）を見出し名に正規化したら、
    調理手順内に登場する同記号も対応する見出し名で置換する
  - 例:「◇のボウル」→「マヨネーズソースのボウル」
  - 例:「◆を加えて」→「漬けダレを加えて」
  - 例:「〇で出てきた水分をとじる」→「水溶き片栗粉で出てきた水分をとじる」
  - 材料側に対応する見出し名が無い場合のみ、記号を残して構わない

### コツ・ポイント
- 温度指示・火加減・時間の Tip を抽出
- PDF 本文や注釈（* 印など）に散在する Tip を箇条書き化
- `・` で列挙、最大 5 項目
- 該当なしなら空欄

### 調理時間（分）
- PDF に明記があればその数字
- 無ければ調理手順から推定（下ごしらえは含めない）
- 最終値は分単位の半角整数。推定なら 5 分刻みで丸める

### 人数分
- PDF ヘッダーまたはタイトル横の「N 人分」を抽出
- 半角整数のみ

### 料理の由来・コラム
- PDF 内の `■` マーク以降の文章を抽出
- 改行・段落構成は維持
- 整形は最小限（過剰な編集禁止）

### 原典作成年月
- PDF ヘッダー右上の「YYYY.M」を `YYYY-MM-01` 形式に変換
- 例:「2012.6」→ `2012-06-01`、「2018.5」→ `2018-05-01`
- 不明なら空欄

### slug
- タイトルを ASCII 英小文字ローマ字に変換し、ハイフン区切り
- 例: 海老マヨ → `ebi-mayo`、回鍋肉 → `hoikoro`、
       麻婆豆腐 → `mabo-dofu`、油淋鶏 → `yurinchi`
- 既存 slug と衝突する場合は末尾に `-2` `-3` を付ける

## 触らないプロパティ
以下はシェフが手動で入力するため、エージェントは更新しないこと:
- メイン画像、動画URL、シェフコメント、公開プラン、公開ステータス、
  公開日時、元素材（PDF/画像）

## 完了報告
全プロパティの更新が終わったら、ページ本文に以下を出力:

- カテゴリ: <値>
- 地方・流派: <値>

次のアクション: シェフが上記 3 項目を入力 → 公開ステータスを「公開」に変更
```

## 4. プロパティ別抽出ルール（詳細）

### 4.1 タイトルの正規化

PDF のタイトル行は冗長になりがちなので、3 段階で短縮します。

| Before（PDF原文） | After（DB） | 短縮理由 |
|---|---|---|
| 沙律蝦球(サーリーシャーチュウ)：海老のマヨネーズ和え | 海老マヨ | 通称優先 |
| さっぱり☆回鍋肉 | 回鍋肉 | 装飾語除去 |
| 鍋+本締で 熱々麻婆豆腐 | 麻婆豆腐 | 副題除去 |
| 油淋鶏;ユーリンチ;サクサク鶏の唐揚香味ソース | 油淋鶏 | 主名優先 |

カナ・原語・別名は **サブタイトル・読み仮名** に逃がす。

### 4.2 材料グループの整形例

```
【主材料】
・海老（大きめ）12 尾（ブラックタイガー13/15）
・塩・白こしょう 少々
・酒 小さじ2
・卵白 小さじ2
・片栗粉 大さじ4
・レタス 4 枚
・揚げ油 適量

【マヨネーズソース】
・マヨネーズ 大さじ4（60g）
・コンデンスミルク 大さじ1（20g）
・ケチャップ 大さじ1（20g）
・レモン果汁 大さじ1/2
・生クリーム 大さじ1
```

### 4.3 調理手順のサブセクション統合 ＋ 記号置換

油淋鶏のように `<香味ソースを作る>` `<鶏肉を揚げる>` `<仕上げ>` と細分化されている場合（**◇ は材料側で「香味ソース」と見出し化された前提**）:

```
1. ①香味ソース：小鍋に合わせた香味ソースを火にかけ、沸いてきたところでねぎ・生姜を入れ、温まれば火を止める
2. ②鶏肉を揚げる：つけ込んだ鶏肉に、まんべんなく片栗粉をまぶし、中温170℃に熱した揚げ油に入れ、火が通るまで揚げる
3. ②鶏肉を揚げる：最後は高温にし、サクサクになるようにからりと揚げ、網バットに取り出す
4. ③仕上げ：お皿にレタスを敷き、2 を 1cm 幅にカットしてのせ、熱々の香味ソースをかけて頂きます
```

海老マヨの例（**◇ は材料側で「マヨネーズソース」と見出し化された前提**）:

```
1. エビに卵白をからめ、片栗粉をはたく
2. 高温180〜190℃に熱した揚げ油で、エビをからりと揚げる
3. マヨネーズソースのボウルに入れて全体にソースをからめ、レタスの上に盛り付け、熱々を頂きます
```

回鍋肉の例（**◆ は材料側で「合わせ調味料」と見出し化された前提**）:

```
4. 味見をし、良ければ水溶き片栗粉で出てきた水分をとじるようにとろみをつけ、お皿に盛り付ける
```
（元 PDF の「〇片栗粉 〇水」を「水溶き片栗粉」に置換）

## 5. エッジケース

| ケース | エージェントの振る舞い |
|---|---|
| 1 PDF に複数レシピ（メイン+副菜+デザート等） | 最初に登場する主役レシピのみ抽出。残りは本文末尾に「同 PDF 内：◯◯／◯◯」とメモ |
| PDF が画像ベース（OCR が必要） | Notion AI の OCR を信頼。読み取れない箇所は空欄＋本文に「OCR読取困難：◯◯」と注釈 |
| 由来・コラムが無い | 該当プロパティを空欄。代わりに Wikipedia 等で補完しない |
| 中華以外の料理（PDFが将来増える想定） | 該当する地方・流派を選ぶ。該当無しなら空欄 |
| 数量に表記揺れ（「大さじ１」「大1」） | 「大さじ1」に統一 |
| 同じレシピで slug 衝突 | 末尾に `-2` `-3` を付ける（DB の既存値は AI に提示しないので、シェフ校正で対処） |
| 添付ファイルが PDF 以外（写真メモなど） | 画像読み取りで可能な限り抽出。情報量が少ない場合は本文に「写真メモのみのため要追記」と注釈 |

## 6. 校正チェックリスト（シェフ用）

エージェント完了後、シェフが各行をレビューする際の観点。

- [ ] **タイトル** が短く実用的か（カナ・原語が混入していないか）
- [ ] **概要** が 80-120 字に収まっているか、誇大表現が無いか
- [ ] **材料** のグループ見出しが妥当か（◇◆〇 が正しく分類されているか）
- [ ] **下ごしらえ** と **調理手順** が混ざっていないか
- [ ] **コツ・ポイント** が箇条書きとして読みやすいか
- [ ] **調理時間** の推定が現実と乖離していないか
- [ ] **由来・コラム** が PDF 原文の意図を保っているか
- [ ] **メイン画像** を選び、添付したか
- [ ] **動画URL** （あれば）を貼ったか
- [ ] **シェフコメント** を一言追記したか
- [ ] **slug** が URL 用に適切か（短く・意味が通る）

## 7. Notion 設定手順

### 7.1 カスタムエージェントを作る

1. Notion 右上の `AI` アイコン → `エージェントを管理` → `新しいエージェント`
2. 名前: `レシピ抽出くん`
3. アイコン: 🍳
4. 説明: 「レシピ PDF を読んで DB に書き戻す」
5. **Instructions**: 本書 §3 の長文プロンプトをそのまま貼り付け
6. **Data sources**: レシピ DB（`7c10d3b1-b478-450f-bece-a5eeb6a05f88`）を接続
7. **Capabilities**:
   - [x] PDF 読み取り
   - [x] 画像読み取り（OCR）
   - [x] データベースのプロパティ更新

### 7.2 起動方法

レシピ DB の任意の行を開いて、以下のいずれか:

- **A 案（推奨）**: ページ本文で `@レシピ抽出くん 抽出して` と入力
- **B 案**: ページ右上の AI ボタン → エージェント一覧から `レシピ抽出くん` を選択
- **C 案（将来）**: DB に「抽出ボタン」プロパティを追加 → ボタンのアクションでエージェント呼び出し

### 7.3 連続バッチ実行（複数 PDF を一気に処理）

シェフ用バッチ手順:

1. レシピ DB に複数行を新規追加
2. 各行の「元素材」プロパティに 1 PDF ずつドロップ
3. 各行を順に開いて `@レシピ抽出くん 抽出して` を実行
4. 全行のレビューを最後にまとめて実施

> **注意**: Notion AI のレート制限により、連続 5-10 件を超えると一時的に処理が遅くなることがあります。

## 9. 既知の制約・注意点

| 項目 | 制約 | 対策 |
|---|---|---|
| メイン画像の自動設定 | Notion AI はファイル添付プロパティに直接ファイルを書き込めない（仕様） | シェフが手動で PDF 内の完成写真をスクリーンショット → 添付 |
| ボタン型プロパティの API 作成 | Notion API では作成不可 | UI で手動追加（Phase 1 の引き継ぎ事項） |
| 工程画像の抽出 | PDF 内の複数枚の調理写真を切り出して別ファイルとして添付するのは AI には困難 | 工程画像プロパティは Phase 2 では未対応。必要なら Phase 4 でツール（Python+pdfplumber 等）を別途構築 |
| 多言語 PDF | 中華以外の英語・フランス語 PDF が来た場合、抽出ルールは英語版を別途用意する必要あり | 当面は日本語 PDF のみ対象 |
| AI の幻覚 | 推定値（特に調理時間）に誤差あり | 校正チェックリスト §6 で必ず確認 |

## 11. 次フェーズへの引き継ぎ事項

Phase 3（Notion ボタン → GAS Webhook → Vercel デプロイ）で必要になる前提:

1. **公開ステータス** が「公開」に変更されたタイミングを Notion Automation で検知
2. Automation の Webhook アクションで GAS の Web App URL を叩く
3. GAS は Notion API で該当行を取得し、JSON に整形して GitHub に push
4. Vercel が GitHub の変更を検知して自動デプロイ
5. デプロイ完了後、Notion Automation で **公開日時** にタイムスタンプを書き戻す

→ Phase 3 設計書で詳細化します。
